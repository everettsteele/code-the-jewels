Code The Jewels

A programming language with personality  ·  transpiles to JavaScript

v0.1 RTJ0 MIT license TypeScript compiler Node.js runtime 13 keywords 4 stdlib modules

What this is

Built for the joy of it

Code The Jewels is a transpiled language inspired by Run The Jewels. You write .rtj files. The compiler, written in TypeScript, turns them into clean JavaScript. The runtime ships with a standard library where every module is named after a city. The CLI gives you a REPL. The language has 13 reserved words, a pipe operator, and a two-branch composition construct called duo(). It is weird on purpose. That is the point.

Built as part of Meridian's Creative Shit category: things built for the joy of building them. No MRR target. No maintenance SLA. MIT licensed. Fork it, break it, extend it.

Language reference

13 keywords

Every reserved word maps directly to a JavaScript concept. The language is deliberately small.

jewel→ const / letDeclare a variable. Everything is a jewel.
verse→ functionDefine a function. Every function is a verse.
send→ returnReturn a value from a verse.
talk→ console.logPrint to stdout. Talk is cheap. Use it anyway.
ifwild→ ifConditional branch. Things get wild when true.
elsewild→ elseThe other branch. Still wild.
run→ forLoop. Run it back.
in→ in (loop)Loop target. Pairs with run.
feature→ importImport a module. Feature the homies.
from→ fromModule source. Pairs with feature.
yank→ throwThrow an error. Yank the chain.
duo→ compositionTwo-branch pipeline. mike runs first, el second.
mike / el→ duo branchesBoth required inside a duo() block.

|> Pipe operator

Thread a value through a chain. Each output becomes the next input. Compiles to nested function calls.

.rtj 
jewel result = "  hello world  "
  |> trim
  |> upper
  |> split(" ")

// compiles to: split(upper(trim("  hello world  ")), " ")

duo() — the signature construct

mike runs first, el runs second. Both are required. The output chains between them.

.rtj 
jewel result = duo(inputValue) {
  mike: processFirst
  el:   processSecond
}

// compiles to: processSecond(processFirst(inputValue))

Real code

Show me

Five programs. Hello world to composition.

1 — hello world

.rtj
talk "Run The Jewels forever"

2 — variables and functions

.rtj
jewel trackName = "Legend Has It"
jewel bpm       = 118

verse describe(name, tempo) {
  send `${name} hits at ${tempo} BPM`
}

talk describe(trackName, bpm)

3 — pipe chain

.rtj
feature { trim, upper, split } from "bk:text"

jewel words = "  tom morello  "
  |> trim
  |> upper
  |> split(" ")

talk words  // ["TOM", "MORELLO"]

4 — loop and conditional

.rtj
jewel tracks = ["Legend Has It", "Talk to Me", "Yankee"]

run track in tracks {
  ifwild (track.length > 10) {
    talk `Long: ${track}`
  } elsewild {
    talk `Short: ${track}`
  }
}

5 — duo() composition

.rtj
feature { count } from "atl:data"
feature { words } from "bk:parse"

jewel line = "Stay gold Ponyboy stay gold"

jewel result = duo(line) {
  mike: words
  el:   count
}

talk result  // 5

Standard library

Named after cities

Four modules. Brooklyn handles text. Atlanta handles data. Every name is a city in the RTJ universe.

bk:textBrooklyn
trim(str) — strip whitespace
upper(str) — to uppercase
lower(str) — to lowercase
split(str, sep) — split to array
join(arr, sep) — join to string
bk:parseBrooklyn
words(str) — split into words
lines(str) — split into lines
chars(str) — split into chars
atl:dataAtlanta
count(arr) — array length
countBy(arr, fn) — conditional count
first(arr) — first element
last(arr) — last element
atl:flowAtlanta
identity(x) — returns x unchanged

Command line

rtj

Four commands. Run it, compile it, check it, or drop into the REPL.

rtj run file.rtjExecute a .rtj file directly. Transpiles to JS in memory and runs immediately.
rtj compile file.rtjTranspile to output.js. Writes generated JavaScript to disk.
rtj check file.rtjSemantic analysis without execution. Reports type and scope errors.
rtj replInteractive REPL. Prompt shows rtj>. Type .exit to quit.

Compiler architecture

How it works

Six stages, all named after RTJ references. The Fist parses. The Hands define the AST. Mike and El run analysis passes. The Gun generates output. The Jewel Box is the runtime.

The HandsAST
→
Mike PassSemantic

El PassLowering
→
The GunJS Generator
→
Jewel BoxRuntime + stdlib
